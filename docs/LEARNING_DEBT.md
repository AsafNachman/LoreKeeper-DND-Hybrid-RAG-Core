| Background Warmup (Threading) | Offloading expensive initialization to a daemon thread so the UI renders instantly while the engine builds. |
| VRAM Saturation vs. Compute | Trade-off between model size (VRAM footprint) and inference quality; smaller quantized models free GPU memory for larger batch sizes or concurrent requests. |
| Model Quantization (4-bit) | Reducing weight precision (FP16 → INT4) to shrink model size and VRAM usage with minimal quality loss; used by phi3 and Q4_K_M GGUF variants. |
| Non-blocking UI Refresh | Polling-based `st.rerun()` loop that periodically re-renders the Streamlit page until a background condition (e.g., engine ready) is met. |
| VRAM Offloading | Splitting model layers between GPU VRAM and CPU RAM when the full model exceeds VRAM capacity; Ollama handles this automatically via `num_gpu` layers. |
| Hallucination Mitigation (small LLMs) | Prompt techniques for constrained small models: explicit "not found" fallback, hard bans on fabrication, per-claim citation enforcement. |
| NVIDIA Container Toolkit Configuration | `NVIDIA_VISIBLE_DEVICES`, `NVIDIA_DRIVER_CAPABILITIES`, and `deploy.resources.reservations.devices` in Compose to expose GPU and required driver capabilities to containers. |

| Race Conditions in Streamlit Session State | Streamlit reruns are stateless: values set in one rerun are invisible to background threads and vice-versa. Thread-safe process-level dicts + `threading.Event` are the correct coordination primitives; `st.session_state` is only safe on the main UI thread. |
| Exhaustive vs. Extractive Summarization | Extractive methods copy source sentences verbatim (high fidelity, longer output); abstractive LLMs compress and may lose numeric specifics. For rules RAG, extractive prompting ("Do not summarize — Extract") preserves DCs, costs, and exception clauses that summarization routinely drops. |
| CUDA Device Mapping in Docker | `NVIDIA_VISIBLE_DEVICES=all` (env) and `count: all` (Compose reservation) must both be present; partial configs (e.g., `count: 1` with `VISIBLE_DEVICES=all`) can mismatch what the kernel exposes vs. what Ollama sees. `count: all/2` is an invalid YAML value — must be a plain integer or `all`. |
